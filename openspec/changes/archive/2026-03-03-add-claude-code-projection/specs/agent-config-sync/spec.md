## ADDED Requirements

### Requirement: Claude Code Built-in Projection Target

The sync system SHALL support `claude-code` as a built-in projection target that
automatically maps hierarchical wai skills to Claude Code command files with
translated frontmatter.

#### Scenario: Built-in target activation

- **WHEN** a projection defines `target: claude-code`
- **THEN** the system treats it as a built-in target, not a filesystem path
- **AND** scans `skills/<category>/<action>/SKILL.md` for all hierarchical skills

#### Scenario: Skill file generated at correct path

- **WHEN** `wai sync` processes a `claude-code` target
- **AND** a skill exists at `skills/issue/gather/SKILL.md`
- **THEN** the system writes `.claude/commands/issue/gather.md`
- **AND** creates the `.claude/commands/issue/` directory if it does not exist

#### Scenario: Frontmatter translation

- **WHEN** a wai SKILL.md has frontmatter `name: issue/gather` and
  `description: "Gather issues from codebase"`
- **THEN** the generated Claude Code file has frontmatter:
  ```yaml
  name: "Issue: Gather"
  description: "Gather issues from codebase"
  category: "Issue"
  ```
- **AND** the skill body is written unchanged after the frontmatter block
- **AND** the `name` field is derived by capitalising the first letter of each path
  segment and joining with `: ` (e.g., `issue/gather` → `Issue: Gather`)

#### Scenario: Auto-generated header

- **WHEN** a Claude Code file is generated by `wai sync`
- **THEN** the `---` frontmatter delimiter is on line 1 of the generated file
  (required for frontmatter parsers that enforce strict position)
- **AND** the file body (after the closing `---` of the frontmatter) begins with:
  `<!-- Auto-generated by wai — do not edit directly -->`
- **AND** the source path is noted (e.g., `<!-- Source: skills/issue/gather/SKILL.md -->`)
