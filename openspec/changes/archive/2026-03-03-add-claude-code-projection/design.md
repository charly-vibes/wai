## Context

Claude Code reads agent skill commands from `.claude/commands/<category>/<action>.md`.
wai stores skills in `.wai/resources/agent-config/skills/<category>/<action>/SKILL.md`.
The formats differ in frontmatter field names and expected structure. Today there is no
automated path from one to the other.

## Goals / Non-Goals

- Goals:
  - Zero-maintenance sync from wai skills to Claude Code commands
  - Correct frontmatter translation (wai → Claude Code format)
  - Works with flat names (`my-skill`) as a no-op or single-level command
- Non-Goals:
  - Bidirectional sync (Claude Code → wai)
  - Support for Cursor, Windsurf, or other tools (separate future change)
  - Watching for changes and syncing automatically

## Decisions

- Decision: `claude-code` is a named built-in target, not a strategy
  - Detecting `target == "claude-code"` before the strategy dispatch keeps
    `strategy` a separate orthogonal axis; the built-in target has its own
    internal strategy (generate)
  - Alternative: add `strategy: generate` with `format: claude-code` — rejected
    as more verbose with no benefit at this scope

- Decision: Frontmatter translation is deterministic, not configurable
  - `name`: `"<category>: <action>"` (capitalised from path segments)
  - `category`: capitalised category segment
  - `description`: copied verbatim
  - No other fields are emitted
  - Alternative: user-configurable mapping — deferred, YAGNI for now

- Decision: Generate files (copy with header + transformed frontmatter), not symlinks
  - Claude Code reads the file format literally; a symlink to SKILL.md would
    require Claude Code to parse wai frontmatter, which it cannot do
  - Generated files are safe to overwrite on every `wai sync`

## Risks / Trade-offs

- Generated file divergence: if the user manually edits `.claude/commands/`,
  changes are silently overwritten on next `wai sync`. Mitigated by the
  `# Auto-generated by wai` header warning.
- Flat skills (non-hierarchical): no `category` prefix; generated at
  `.claude/commands/<name>.md` with `name: <Name>` and no `category` field.
  Acceptable for backward compatibility.
