Root cause of 'wai prime' not connected error: In resolve_project() in prime.rs, close.rs, and add.rs — when multiple projects exist and --no-input is not set, code falls through to cliclack::select().interact(). In non-TTY environments (Claude Code bash tool, pipes, CI), cliclack fails with 'not connected'. Fix: guard with both ctx.no_input || \!std::io::stdin().is_terminal() so that non-TTY environments also get the clear NonInteractive error with hint to pass --project <name>. Affected files: src/commands/prime.rs, src/commands/close.rs, src/commands/add.rs. std::io::IsTerminal available since Rust 1.70, using Rust 1.93.
