Bug found and fixed in wai why git integration: gather_git_file_context and gather_recent_commits were running git commands without .current_dir(project_root), so they used the process CWD instead of the repo root. This caused two problems: (1) relative file paths like src/main.rs were resolved from CWD, not project root, so git log would find nothing when invoked from a subdirectory; (2) recent commits could fail if git wasn't in scope. Fix: pass project_root to both functions and set .current_dir(project_root). Also canonicalize the file path (resolving from process CWD), then make it relative to project_root so git sees it correctly regardless of where wai was invoked.
