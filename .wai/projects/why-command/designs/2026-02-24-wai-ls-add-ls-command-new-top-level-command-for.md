wai ls (add-ls-command): new top-level command for cross-project global view. Design decisions: (1) scan strategy = walk from HOME depth=3 by default, --root and --depth flags for override, skip hidden dirs except .wai/ — zero-setup, no global registry; (2) performance = depth limit fast enough for MVP, no caching; (3) beads counts = invoke bd stats --json per workspace when .beads/ detected, skip gracefully if bd unavailable; (4) output = one line per (workspace, project), columns padded to longest name, counts column omitted when no workspace has beads. Openspec: openspec/changes/add-ls-command/.
